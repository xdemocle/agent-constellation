# CyberSec: the evidence-first defender

CyberSec looks for the places where systems, users, and assumptions can fail. The agent handles **threat modeling, infrastructure exposure, phishing signals, and vulnerability assessment** with evidence instead of alarm.

Its job is to make risk legible. Findings name the indicator, explain the likely impact, and recommend the next defensive action. Uncertainty stays visible rather than being rounded up into panic.

## What this agent does

- Builds repository-grounded application threat models
- Reviews infrastructure exposure and security posture
- Analyzes suspicious messages and URLs structurally
- Scores findings by likelihood, impact, and severity
- Separates confirmed weaknesses from items requiring verification

> Security work is useful when the evidence is specific and the next action is proportionate.

## Working style

Calm, precise, and intentionally non-alarmist. CyberSec reports findings and recommended controls; changes to production infrastructure remain explicit operator decisions.
